## 设计目标
langchain4j中大模型主要分为：聊天模型、图像模型、向量模型三类
其余模型可以通过这几种基础模型组合实现特有功能

## RAG
RAG 原理是将本地存储的数据与用户询问的提示词一起传递给AI，让AI处理数据，可以尽量杜绝掉模型幻觉的此情况。
RAG 的实现原理：
1. 创建一个向量数据库，将本地存储的数据进行向量化，并保存到向量数据库中。
2. 使用向量模型将用户询问的提示词进行向量化，并查询向量数据库，获取最相似的向量。
3. 将最相似的向量与用户询问的提示词一起传递给AI，让AI处理数据。

## MCP 和 工具
目前大部分 chat 模型支持工具调用，当用户提示词中出现关键词时，系统会工具关键去调用封装的工具@tool
系统根据根据返回的数据与用户提示词一起传递给AI处理，返回
实现原理
1. 创建一个工具类，将工具类封装成工具，并注册到系统。
2. 使用工具类，将工具类封装成工具，并注册到系统。

# 项目概览：Chat Book Cloud (CBC)

**定位：** AI 原生驱动的沉浸式知识创作与精准检索平台。
**核心理念：** “为创作者提效，为读者降噪”。通过 Agent 介入，打破传统博客平台“信息冗余”与“创作低效”的僵局。

## 一、 核心价值主张 (Core Value Prop)

### 1. 读者侧：从“海量检索”到“语义直达”

* **双引擎检索：** 结合 **精确匹配 (Elasticsearch)** 与 **高质量语义匹配 (RAG/Vector Search)**。
* **内容去噪（AI 降噪）：** 针对 BUG 类文章，Agent 自动提取“极简解决方案”；针对技术专栏，Agent 辅助生成“知识地图”和“思路导图”。
* **审美主义 UI：** 采用**简约而不简单 (Sophisticated Minimalism)** 的设计语言。借鉴 CSDN 的专业布局，但剔除冗余广告，强化沉浸式阅读体验，提高用户留存率。

### 2. 作者侧：IDE 级别的创作体验

* **混合动力编辑器：** * **Markdown + 富文本工具栏：** 保持 MD 的高效语法，同时提供可视化的快捷操作。
* **IDEA 级快捷键支持：** 支持 Ctrl+D (复制行)、Ctrl+X (剪切行) 等习惯，降低开发者的迁移成本。


* **创作保障机制：** 引入**实时流式保存 (Auto-save)** 与 **多端离线缓存**，解决断网/断电导致的内容丢失焦虑。
* **AI 协同创作：** * **Text Completion：** 类似 Copilot 的文本补全，预测作者思路。
* **智能校对：** Agent 实时检测逻辑漏洞、语法错误及代码块的正确性。



### 3. 运营与社交侧：垂直出圈与社交粘性

* **垂直领域深耕：** 专注于高质量技术分享，初期通过“极致的 Agent 辅助功能”吸引极客群体。
* **社交闭环：** 集成 **实时聊天 (WebSocket)** 与好友系统，将“阅读-学习-讨论”留在平台内。

---

## 二、 技术架构预想 (Tech Stack)

作为基于 **Spring Cloud** 和 **Vue** 的工程，项目将采用 **DDD (领域驱动设计)** 模式：

* **后端：** Spring Boot 3 + Spring Cloud Alibaba + **Spring AI / Langchain4j** (处理 Agent 逻辑)。
* **存储：** MySQL (业务数据) + Redis (缓存/聊天) + **Vector DB** (向量数据库，用于 RAG 检索)。
* **前端：** Vue 3 + Vite + Tailwind CSS (实现专业布局)。
* **AI 能力：** 集成 **MCP (Model Context Protocol)** 协议，使平台能够通过 Agent 调用外部开发工具或本地知识库。
